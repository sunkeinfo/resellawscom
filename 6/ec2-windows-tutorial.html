AWS EC2 韩国区域 Windows 2022 实例启动教程

⚠️ 重要提示：本教程将创建一个具有完全开放防火墙的EC2实例，仅用于测试环境。生产环境请务必配置适当的安全组规则。

1. 前提条件

  • 拥有有效的AWS账户
  • 具有EC2实例创建权限
  • 了解AWS基本操作界面
  • 确保账户有足够的配额启动C5N.9XLARGE实例

2. 登录AWS控制台

  1. 访问 AWS管理控制台
  2. 使用您的AWS账户凭证登录
  3. 在右上角选择 亚太地区（首尔）ap-northeast-2 区域
[截图：AWS控制台区域选择界面]

3. 启动EC2实例

3.1 进入EC2服务

  1. 在AWS控制台搜索框中输入"EC2"
  2. 点击"EC2"服务进入EC2控制面板
  3. 点击 "启动实例" 按钮

3.2 配置实例详细信息

步骤1：选择AMI（Amazon Machine Image）

  1. 在"名称和标签"部分，输入实例名称：Windows-2022-Test
  2. 在"应用程序和操作系统映像"部分：
    • 选择 "Microsoft Windows"
    • 选择 "Microsoft Windows Server 2022 Base"
    • 架构选择：64位(x86)

步骤2：选择实例类型

  1. 在"实例类型"部分，搜索并选择 c5n.9xlarge
  2. 确认配置：36 vCPUs, 96 GiB 内存
配置项 值 说明
实例类型 c5n.9xlarge 高性能计算优化实例
vCPUs 36 虚拟CPU核心数
内存 96 GiB 系统内存
网络性能 高达100 Gbps 网络带宽

3.3 密钥对配置

  1. 在"密钥对（登录）"部分
  2. 选择 "创建新的密钥对"
  3. 密钥对名称：windows-korea-key
  4. 密钥对类型：RSA
  5. 私有密钥文件格式：.pem
  6. 点击 "创建密钥对"
  7. 系统会自动下载 windows-korea-key.pem 文件，请妥善保存
重要：私钥文件只能下载一次，请务必保存在安全位置。丢失私钥将无法获取Windows管理员密码。

3.4 网络设置

  1. 在"网络设置"部分，点击 "编辑"
  2. VPC：保持默认或选择您的VPC
  3. 子网：选择可用区域内的公有子网
  4. 自动分配公有IP：启用
  5. 防火墙（安全组）：
    • 选择 "创建安全组"
    • 安全组名称：windows-open-all
    • 描述：允许所有流量（仅测试用）

安全组规则配置：

类型 协议 端口范围 源 描述
所有流量 所有 所有 0.0.0.0/0 允许所有入站流量
⚠️ 安全警告：开放所有流量存在严重安全风险，仅适用于测试环境。生产环境请限制特定端口和IP范围。

3.5 配置存储

  1. 在"配置存储"部分
  2. 根卷配置：
    • 大小：50 GiB
    • 卷类型：gp3（通用SSD）
    • IOPS：3000（默认）
    • 吞吐量：125 MiB/s（默认）
    • 加密：可选择启用

4. 启动实例

  1. 检查所有配置是否正确
  2. 在右侧摘要面板中确认：
    • 实例类型：c5n.9xlarge
    • AMI：Windows Server 2022
    • 存储：50 GB gp3
    • 安全组：允许所有流量
  3. 点击 "启动实例" 按钮
  4. 等待实例状态变为"正在运行"（通常需要3-5分钟）

5. 获取Windows管理员密码

5.1 等待实例完全启动

  1. 在EC2控制台中选择您的实例
  2. 等待实例状态变为"正在运行"
  3. 等待状态检查完成（2/2项检查通过，通常需要5-10分钟）

5.2 获取管理员密码

  1. 在EC2控制台中，右键点击您的实例
  2. 选择 "连接"
  3. 点击 "RDP客户端" 选项卡
  4. 点击 "获取密码" 按钮
  5. 点击 "浏览" 选择之前下载的 windows-korea-key.pem 文件
  6. 点击 "解密密码"
  7. 复制显示的管理员密码
[截图：AWS控制台获取Windows密码界面]

6. 连接到实例

6.1 使用远程桌面连接

  1. 打开Windows远程桌面连接（mstsc）
  2. 输入实例的公有IP地址
  3. 使用以下凭证登录：
    • 用户名：Administrator
    • 密码：从AWS控制台获取的解密密码
  4. 点击"连接"

6.2 替代连接方法

您也可以直接从AWS控制台连接：

  1. 在实例详情页面，点击 "连接"
  2. 选择 "RDP客户端"
  3. 点击 "下载远程桌面文件"
  4. 双击下载的.rdp文件
  5. 输入解密后的管理员密码

7. 验证配置

登录后验证以下项目：

  1. 系统信息：验证系统为Windows Server 2022，硬盘容量为50GB
  2. 网络连接：确认可以访问互联网
  3. 管理员权限：确认当前用户具有管理员权限

验证命令（在PowerShell中执行）：

# 检查系统信息 Get-ComputerInfo | Select-Object WindowsProductName, TotalPhysicalMemory # 检查磁盘空间 Get-WmiObject -Class Win32_LogicalDisk | Select-Object DeviceID, Size, FreeSpace # 检查网络连接 Test-NetConnection -ComputerName google.com -Port 80

9. 安全建议

⚠️ 重要安全提醒

  • 更改管理员密码：登录后建议更改Administrator密码
  • 限制网络访问：生产环境中应限制安全组规则，仅开放必要端口
  • 启用CloudTrail：监控API调用和用户活动
  • 定期更新：保持Windows系统和软件更新
  • 备份策略：配置EBS快照进行数据备份
  • 监控成本：设置账单警报避免意外费用

10. 故障排除

常见问题及解决方案：

问题1：无法连接到实例

  • 检查实例状态是否为"正在运行"
  • 确认安全组规则允许RDP连接（端口3389）
  • 验证公有IP地址是否正确
  • 等待用户数据脚本执行完成

问题2：无法获取管理员密码

  • 确认实例已完全启动（状态检查通过）
  • 检查私钥文件格式是否正确
  • 等待更长时间后重试（最多15分钟）

问题3：实例启动失败

  • 检查账户配额限制
  • 确认选择的可用区支持c5n.9xlarge
  • 验证IAM权限

11. 清理资源

测试完成后，请及时终止实例以避免持续费用：

  1. 在EC2控制台选择实例
  2. 点击"实例状态" → "终止实例"
  3. 确认终止操作
  4. 删除相关的安全组（如不再需要）
